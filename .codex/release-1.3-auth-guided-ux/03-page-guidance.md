# Page guidance

A centralized page metadata registry defines route ID, pattern, area, bilingual title and summary, purpose, tasks, actions, help, glossary terms, next route, experience visibility, and access class. The shell derives Page Introduction, breadcrumbs, Help context, Assistant orientation, tours, search metadata, and analytics category from it.

Introductions stay concise and provide purpose, useful outcome, primary/secondary actions, availability where relevant, and a task-oriented Help link. Shared empty states describe what is missing, why it matters, and the first safe action.

## Acceptance

Primary and nested routes resolve metadata, breadcrumb parents are valid, and Help/Assistant wording derives from structured bilingual content.
