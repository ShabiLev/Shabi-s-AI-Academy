# Profile and account

`/profile` stores optional display names, learning goals, interests, language, experience mode, onboarding preference, and data shortcuts. Guest profiles remain browser-local. Internal cloud UUIDs are not prominent outside Developer Mode.

`/account/security` requires an authenticated cloud account and explains email verification, password recovery/change, session information, sign-out, export, and deletion limitations. Delete-account UI requires typed confirmation and does not claim backend deletion is complete: administrative deletion needs a trusted server operation outside this release.
