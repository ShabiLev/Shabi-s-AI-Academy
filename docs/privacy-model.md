# Privacy model

The Academy is local-first. Guest learning progress, prompts, agents, projects, knowledge, workflows, runs, preferences, onboarding, and analytics remain in the current browser unless the user configures, signs in, and explicitly chooses a cloud operation.

Supabase may process account identifiers, email, profile preferences, and selected user-owned records. Passwords and authentication sessions are provider-managed and excluded from Academy exports. Built-in catalogs contain no user data. Local analytics are optional and are not sold or sent to an external analytics provider in this beta.

Users can export local workspace data and clear selected local domains. Cloud deletion requires authenticated backend support; the UI states when only a request can be recorded. See `/privacy` for the bilingual beta notice. This documentation is engineering guidance, not legal advice.
