# Guided onboarding

`/onboarding` is a seven-step, bilingual introduction for a first Guest session or newly registered user. It explains local/cloud boundaries and Mock/Dry Run limitations, records a main goal, experience level, and interests, then chooses a deterministic starting recommendation.

The recommendation uses only the stored selections; it is not model-generated. The final step updates the experience mode and opens `/dashboard`. Users can continue as Guest, register, or sign in. Profile and Settings can restart onboarding later.

Existing browser-local records are not migrated, deleted, or uploaded by onboarding. Substantial existing work is never used as permission to start cloud synchronization.
