# Navigation

One typed navigation model owns labels, icons, product areas, route targets, and experience visibility. Desktop and mobile render the same hierarchy. Group expansion is a bounded local UI preference. Active routes remain selected, labels accompany every icon, and the sidebar exposes only one scroll region.

Breadcrumbs and the Header consume the page metadata registry rather than maintaining separate route-title maps. Detail routes resolve to valid list parents and use logical separators in RTL/LTR.

## Acceptance

Group state persists safely, labels remain visible with icons, and all viewports retain one understandable navigation hierarchy.
