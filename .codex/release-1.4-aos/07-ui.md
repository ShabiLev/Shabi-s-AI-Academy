# UI

The authenticated `/aos/progress` and `/aos/memory` routes expose current task, requirement-derived progress, blockers, evidence freshness, research counts, next actions, and handoff state from the sanitized Pages-safe snapshot. Raw paths, logs, secrets, and environment values are excluded.

The protected `/aos` dashboard and its `modules`, `research`, `evidence`,
`handoffs`, `security`, and `releases` subroutes are bilingual, responsive, and
lazy-loaded. They present generated metadata only and label absent or stale state
honestly.

All routes require Hebrew RTL and English LTR parity, semantic headings and tables,
keyboard focus, visible controls, safe wrapping for identifiers and paths, and no
primary-content horizontal overflow from 320px through desktop and at 200% zoom.
Navigation, page inventory, help metadata, breadcrumbs, and active state must stay
consistent.
