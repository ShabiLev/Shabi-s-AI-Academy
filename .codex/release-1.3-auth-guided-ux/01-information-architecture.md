# Information architecture

The application shell organizes routes into Home, Learn, Build, Workspace, and More. Home holds the task-oriented Dashboard; Learn holds Journey, Lessons, and Radar; Build holds personal libraries, Playgrounds, and Workflows; Workspace holds Projects, Knowledge, and Run History; More holds orientation, settings, and progressively disclosed diagnostics.

Beginner is the default for new preferences. It keeps every route addressable but simplifies visible navigation and page copy. Advanced reveals analytics, QA, documentation, release and runtime detail. Developer Mode requires an additional explicit setting. The preference is versioned browser-local data and is not an authorization role.

## Acceptance

Each route appears in one group, both shells share the model, and switching modes never changes route or data permissions.
