# Version 1.5 Capability Registry

## Contract

The registry validates capabilities before registration and supports bounded search by availability, risk, and online/offline mode. Records declare owner, version, inputs, outputs, permissions, dependencies, provider, cost, latency, health, and internal documentation.

## Initial capabilities

The initial catalog documents the AOS Event Bus, Scheduler, and cached Radar feed. It describes implemented availability and does not fabricate tools, provider connections, models, MCP state, or credentials.

## Extension rule

New capabilities require a stable identifier, semantic version, accountable owner, explicit permissions and dependencies, a valid internal documentation route, and validation/discovery tests.
